# 🎮 Cyber Arena — Система бронирования игровых зон

Система управления бронированием компьютерных зал и игровых зон для колледжа.

## 📋 Описание

**Cyber Arena** — это веб-приложение для бронирования игровых мест (ПК, консоли, командные комнаты). Проект разработан для хакатона и демонстрирует полный стек технологий: от базы данных до пользовательского интерфейса.

### Основные возможности

- ✅ Регистрация и авторизация пользователей (JWT)
- ✅ Просмотр доступных зон кибер-арены
- ✅ Бронирование с проверкой пересечений
- ✅ Календарь занятости
- ✅ Личный профиль с историей бронирований
- ✅ Система достижений
- ✅ Админ-панель для управления зонами и бронированиями

## 🔧 Установка и запуск

### Требования

- Python 3.11+
- PostgreSQL 15+ (или Docker)
- Node.js (опционально, для разработки)

### Шаг 1: Запуск базы данных

**Вариант A: Docker (рекомендуется)**

```bash
docker-compose up -d
```

**Вариант B: Локальный PostgreSQL**

```sql
CREATE DATABASE hahaton;
```

### Шаг 2: Настройка окружения

```bash
cd backend
cp .env.example .env
```

Отредактируйте `.env` при необходимости.

### Шаг 3: Установка зависимостей

```bash
cd backend
pip install -r requirements.txt
```

### Шаг 4: Запуск сервера

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Сервер запустится на `http://localhost:8000`

## 🔐 Проверка пересечения бронирований

Ключевая логика реализована в `backend/app/services/booking_service.py`:

```python
# Условие пересечения: start1 < end2 AND end1 > start2
Booking.start_time < end_time,
Booking.end_time > start_time,
```

**Гарантии:**
- Нельзя забронировать уже занятое время
- Пользователь не может иметь пересекающиеся брони
- При обновлении проверяется конфликт с другими бронями

## 🐛 Возможные проблемы

### Ошибка подключения к БД
```
Error: Could not connect to database
```
**Решение:** Убедитесь, что PostgreSQL запущен и параметры в `.env` верны.

### Ошибка импорта asyncpg
```
ModuleNotFoundError: No module named 'asyncpg'
```
**Решение:** `pip install -r requirements.txt`

### Порт 8000 занят
```
Error: Address already in use
```
**Решение:** Запустите с другим портом: `uvicorn app.main:app --port 8001`
